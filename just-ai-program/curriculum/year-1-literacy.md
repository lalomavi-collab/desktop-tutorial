#### 45 דקות | מסגרת מושגית — "מה AI עושה ומה הוא לא עושה"

**AI — מה הוא:**
- מערכת שלומדת מדוגמאות ומייצרת תחזיות/תוצאות
- אינה "חושבת", אינה "מבינה" — היא מחשבת הסתברויות

**AI — מה הוא אינו:**
- אינו Oracle שיודע אמת מוחלטת
- אינו מחליף שיפוט מקצועי
- אינו "ניטרלי" — משקף הטיות שבנתונים ובמתכנניו
- אינו יציב ללא הכוונה — "Garbage in, Garbage out"

**שלושה דורות של AI — פרספקטיבה היסטורית:**
```
1950-1980  │  AI מבוסס כללים (Rule-Based)
           │  אם X אז Y — הגיון מפורש
           │  דוגמה: מערכות מומחה בדיני מס
           ▼
1980-2012  │  Machine Learning קלאסי
           │  לומד מדוגמאות — פרמטרים סטטיסטיים
           │  דוגמה: פילטרי ספאם, ניתוח אשראי
           ▼
2012-היום  │  Deep Learning / Foundation Models
           │  למידה היררכית; LLMs (GPT, Claude)
           │  דוגמה: ChatGPT, Copilot, כלי Legal AI
```

### 2.1 — יסודות Prompt Engineering (4 שעות)

#### מסגרת CLEAR לניסוח פרומפטים משפטיים

```
C — Context    │ הגדר הקשר: "אני עורך דין מסחרי בפירמה ישראלית,
               │ מייצג חברת SaaS בסכסוך חוזי עם לקוח ארגוני"
               │
L — Length     │ הגדר אורך: "בלא יותר מ-500 מילה" / "בנקודות"
               │
E — Examples   │ תן דוגמה: "כמו הניסוח בסעיף X שאצרפם"
               │
A — Audience   │ הגדר קהל: "בשפה שתהיה ברורה לשופט עסקי"
               │
R — Role       │ הגדר תפקיד: "הצג כסנגור / כמבקר / כשופט ניטרלי"
```

#### 10 טכניקות מתקדמות

| # | טכניקה | תיאור | דוגמה |
|---|--------|--------|--------|
| 1 | **Role Prompting** | הפוך AI לתפקיד מוגדר | "פעל כעורך דין בקיא בדיני עבודה ישראלי" |
| 2 | **Chain-of-Thought** | בקש הסבר שלב-אחר-שלב | "נתח את הסעיף שלב אחר שלב לפני מסקנה" |
| 3 | **Few-Shot** | תן דוגמאות לפני המשימה | "הנה 2 ניסוחים שאהבתי... עכשיו כתוב בסגנון זה" |
| 4 | **Devil's Advocate** | בקש ניתוח ביקורתי | "מה הטיעונים הטובים ביותר נגד העמדה שלי?" |
| 5 | **Structured Output** | בקש פורמט מסוים | "החזר JSON עם: risk_level, key_clauses, red_flags" |
| 6 | **Iteration** | שפר בסיבובים | "כתוב → בקר → שפר → בקר שוב" |
| 7 | **Negative Prompting** | הגדר מה לא לעשות | "אל תכלול ייעוץ בנושאי מס; אל תציין שמות" |
| 8 | **Temperature Control** | בקש עקביות/יצירתיות | "ענה בצורה שמרנית וזהירה ללא השערות" |
| 9 | **Source Citation** | בקש הפניות | "ציין לכל טענה: 'מקור: [סעיף X / פסיקה Y]'" |
| 10 | **Meta-Prompting** | בקש Prompt טוב יותר | "עזור לי לנסח שאלה טובה יותר כדי לקבל..." |

#### סדנה: "מרתון Prompting" (90 דקות)

**אתגר 1: Due Diligence חוזה שכירות** (30 דקות)
- קלט: חוזה שכירות מסחרית (8 עמודים)
- משימה: כתבו Prompt שיזהה Red Flags, עמידה ב-GDPR, סיכוני יציאה
- השוו תוצאות בין גרסאות Prompt שונות

**אתגר 2: ניסוח עמדת לקוח** (30 דקות)
- תרחיש: לקוח (חברת סייבר ישראלית) נדרש להגיב לבקשת מידע של רגולטור אמריקאי
- משימה: Prompt לניסוח תגובה שמגנה על אינטרסים תוך עמידה ב-SEC disclosure rules

**אתגר 3: Prompt לניתוח פסק דין** (30 דקות)

### 2.2 — Prompt Engineering מתקדם: שיטות לפחות-שגיאות (6 שעות)

#### 2.2.1 — "Prompt לביקורת עצמית" — RAG ו-Grounding

**בעיית ההזיות — פתרון מבני:**

```
בלי Grounding:         עם Grounding (RAG):
שאלה → LLM → תשובה    שאלה → מסמכים שלי → LLM → תשובה
         ↑                              ↑
    מסתמך על            מסתמך על המסמכים שסיפקתם
   "זיכרון" כללי                + בסיס ידע כללי
```

**Prompt Pattern לביקורת עצמית:**
```
שלב א: "ענה על השאלה הבאה בהתבסס *אך ורק* על המסמך שאצרף"
שלב ב: [הדבק מסמך]
שלב ג: [שאלה]
שלב ד: "בסיום, ציין בפורמט: CONFIDENCE: [1-5] ו-LIMITATIONS: [מה לא יכולת לענות]"
```

#### 2.2.2 — Prompt לתחומי מומחיות ספציפיים

**תחום: M&A Due Diligence**
```
תפקיד: "אתה עורך דין M&A בכיר עם 20 שנות ניסיון"
משימה: "סקור את חוזה הרכישה הבא וזהה:"
תוצרים:
- Material Adverse Change clauses + פרשנות סיכונים
- Representations & Warranties חסרות/חלשות
- Indemnification caps ו-baskets — השוואה לשוק
- Regulatory approvals הנדרשים + זמן ריאלי
- Integration risks משוקעים בניסוח
פורמט: "טבלה עם: סעיף | סיכון | חומרה (1-5) | המלצה"
```

**תחום: ליטיגציה**
```
תפקיד: "אתה שופט ניסיון המעריך כתב תביעה"
משימה: "הערך את כתב התביעה הבא מנקודת המבט של:"
1. עילות תביעה — האם מבוססות משפטית?
2. עובדות — האם מפורטות מספיק לעמידה ב-Iqbal/Twombly?
3. נזקים — האם מוכחים ומספקים?
4. חולשות ניכרות מהבוחן הנגדי
5. המלצות שיפור (3 הכי קריטיות)
```

**תחום: ציות ורגולציה**
```
תפקיד: "אתה CCO (Chief Compliance Officer) ב-Fortune 500"
הקשר: [תאר את הארגון, התחום, המדינות הרלוונטיות]
משימה: "לאור הרגולציות הבאות [EU AI Act / GDPR / CCPA], בחן:"
- פערי ציות בתיאור המערכת שאצרף
- פעולות מיידיות (0-3 חודשים)
- פעולות טווח בינוני (3-12 חודשים)
- סיכוני אכיפה אם לא פועלים
פורמט: "Traffic light: 🔴 קריטי | 🟡 דחוף | 🟢 מומלץ"
```

### 4.1 — Client Data בעידן ה-AI (5 שעות)

#### חמשת הסיכונים הקריטיים

**סיכון 1: אימון מחדש (Training on Inputs)**
> האם הפרומפט שלי מאמן את המודל? האם מידע לקוח "נשמר"?
- OpenAI, Anthropic, Google — מדיניות שונות
- Enterprise tiers vs. Consumer plans
- **כלל מעשי**: לעולם אל תציינו שמות לקוחות, מספרי תיק, מזהים ייחודיים

**סיכון 2: Data Residency**
> היכן מאוחסנים הנתונים? האם זה מתאים ל-GDPR / חוק הגנת הפרטיות הישראלי?
- מודל: EU Data Region (Microsoft, Google, AWS)
- ישראל כ-"Third Country" — הגנות נדרשות

**סיכון 3: Confidentiality ו-Attorney-Client Privilege**
> האם שיתוף מידע עם AI מהווה ויתור על פריבילגיה?
- ניתוח: "Third-Party Doctrine" בהקשר AI
- המלצה: Anonymization + Enterprise Agreement

**סיכון 4: Breach בצד הספק**
> מה קורה לנתוני לקוחותיך אם ה-AI vendor נפרץ?
- DPA — Data Processing Agreement — מה לדרוש
- SOC 2 Type II, ISO 27001

**סיכון 5: Insider Threat — העובד כנקודת חולשה**
> כיצד מונעים שימוש לא מורשה ב-AI על ידי עובדים?
- מדיניות שימוש ארגונית (AI Use Policy)
- Training & Awareness

#### מסמך ה-AI Use Policy — מסגרת לניסוח

```
[שם הארגון] — מדיניות שימוש ב-AI

1. כלים מאושרים: [רשימה מאושרת בלבד]
2. נתונים מותרים להזנה: [ציבורי, מאונאנים, פנימי לא מסווג]
3. נתונים אסורים: [PII לקוחות, מסמכים חסויים, מידע MNPI]
4. חובת ביקורת: כל פלט AI חייב אישור עורך דין בכיר
5. תיעוד: כל שימוש מתועד ב-[מערכת]
6. הפרות: [סנקציות]
```

### 4.2 — AI ואבטחת סייבר בפירמות משפטיות (5 שעות)

- Adversarial Attacks — כיצד "מבלבלים" מערכת AI
- Prompt Injection — תקיפה דרך הנחיות זדוניות
- Deepfakes בהליך משפטי — ראיות ואימות
- AI לגילוי איומים — כלים לסייבר פירמה
