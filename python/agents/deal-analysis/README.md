# מערכת ניתוח עסקאות נדל"ן, LALUM

מערכת לניתוח כדאיות של עסקאות נדל"ן: קליטת מסמכים, סקירה תכנונית ומשפטית,
מחשבון כדאיות עם מדרגות מס מלאות, ניתוח AI, עסקאות השוואה ודוח מסכם מעוצב.

## התקנה (פעם אחת)

```bash
cd python/agents/deal-analysis
pip install pypdf openpyxl pytest
pip install anthropic          # לניתוח AI ומחקר תכנוני (אופציונלי)
```

לניתוח AI: הגדר משתנה סביבה `ANTHROPIC_API_KEY` (מפתח מ-platform.claude.com).
בלי מפתח, המערכת עובדת בניתוח מילות מפתח.

המערכת מציגה את רשימת העסקאות, בוחרים מספר, הניתוח רץ והדוח נפתח לבד בדפדפן.

## שימוש מתקדם (שורת פקודה)

1. צור תיקייה לעסקה ושים בה את כל המסמכים (PDF, אקסל, CSV, טקסט):
   נסח טאבו, מידע תכנוני, שמאות, הסכם, אישור מימון.

2. אופציונלי: קובץ `deal.json` בתיקייה עם הפרמטרים הכלכליים:

```json
{
  "price": 2800000,
  "expected_value": 4100000,
  "renovation_cost": 450000,
  "other_costs": 85000,
  "equity": 1200000,
  "loan_years": 2.5,
  "is_single_home": false,
  "area_sqm": 95,
  "betterment_levy": 0,
  "address": "הרצל 25 תל אביב",
  "document_urls": []
}
```

3. הרץ:

```bash
python run_deal.py --folder <תיקיית העסקה> --name "שם העסקה"
```

דגלים שימושיים:
- `--address "כתובת"`: מפעיל עסקאות השוואה (מידע נדל"ן ממשלתי) ומחקר תכנוני
- `--fetch-url <קישור>`: מוריד PDF מקישור לתיקייה לפני הניתוח
- `--no-ai`: דילוג על ניתוח AI
- `--price / --value / --equity / --rate / --years / --single-home`: פרמטרים משורת הפקודה

הדוח נשמר כ-`report.html` בתיקיית העסקה.

## מקורות חיצוניים

| מקור | שימוש | דרישה |
|---|---|---|
| מידע נדל"ן (nadlan.gov.il) | עסקאות השוואה, חציון למ"ר, בדיקת סבירות שווי | אינטרנט |
| בנק ישראל | ריבית עדכנית לברירת מחדל של המימון | אינטרנט |
| Claude API | סיווג וחילוץ מסמכים, מחקר תכנוני | ANTHROPIC_API_KEY |

כשל של מקור חיצוני לא עוצר את הניתוח, מוצגת אזהרה וההרצה ממשיכה.
הערה: בסביבת הענן של Claude Code אתרי gov.il חסומים, ההרצה המלאה מיועדת למחשב המקומי.
