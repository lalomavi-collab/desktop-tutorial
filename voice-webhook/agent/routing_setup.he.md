# ניתוב 03-3104959 למענה ה-AI (בהעברת שיחות + פלטפורמה מוכנה)

המסלול שנבחר: פלטפורמה קולית מוכנה (Vapi או Retell) שנותנת מספר משלה ומריצה את
הבוט. אתה מעביר את השיחות מ-03-3104959 למספר הזה, בדיוק כמו שאתה כבר יודע. אין
צורך בשום הגדרת SIP מול הספק.

זרימה:

```
מתקשר > 03-3104959 > (העברת שיחות שאתה כבר עושה) > מספר הפלטפורמה
      > הבוט עונה ומדבר לפי התסריט > בסיום שולח webhook ל-Backend > Supabase
```

---

## שלב א': הקמת הבוט בפלטפורמה (Vapi לדוגמה)

1. פתח חשבון ב-Vapi (או Retell) והשג מספר טלפון מובנה.
2. צור Assistant חדש. השתמש בתבנית `vapi_assistant.json` שבתיקייה הזו:
   - **הוראות המערכת (system prompt)**: הדבק את כל התוכן של
     `lalum_voice_agent.he.md`. זה נותן לבוט את התסריט היציב, הגבולות, וההפניה
     לאפליקציה.
   - **מודל (LLM)**: Claude (Anthropic). כך התשובות באיכות גבוהה ואחידות.
   - **קול (TTS) ותמלול (STT)**: בחר ספק עם תמיכה טובה בעברית. כדאי לבדוק דוגמה
     קולית קצרה לפני שמתחייבים, כי איכות העברית משתנה בין ספקים.
   - **הודעת פתיחה**: משפט הפתיחה מהתסריט.
3. הגדר את כתובת ה-Server / Webhook של ה-Assistant אל
   `https://<host>/webhook/vapi` (או `/webhook/retell` ב-Retell).
4. אם אתה מגדיר סוד webhook, הכנס אותו גם ב-`VOICE_WEBHOOK_SECRET` בסביבת ההרצה
   של ה-Backend. הפלטפורמה תשלח אותו בכותרת, וה-Backend יאמת.

## שלב ב': העברת השיחות

הפעל אצל ספק ה-VoIP שלך העברת שיחות של 03-3104959 אל מספר הפלטפורמה. זה בדיוק מה
שאתה כבר יודע לעשות. שתי נקודות לוודא:

1. **זיהוי המתקשר המקורי**: רצוי שההעברה תעביר את מספר המתקשר המקורי (CLI), כדי
   שה-CRM ירשום את הלקוח הנכון ולא את מספר ההעברה. אם הפלטפורמה מקבלת את המספר
   המקורי, המערכת שלנו כבר יודעת לזהות אותו.
2. **עלות**: אם מספר הפלטפורמה בחו"ל, ההעברה עלולה להיחשב שיחת חוץ. אפשר לבחור
   פלטפורמה עם מספר מקומי, או לקחת זאת בחשבון.

## שלב ג': חיבור ל-Backend

הרץ את שירות ה-voice-webhook עם הסודות (`SUPABASE_URL`,
`SUPABASE_SERVICE_ROLE_KEY`, `ANTHROPIC_API_KEY`, ואופציונלי
`VOICE_WEBHOOK_SECRET`). ודא ש-`/health` עונה, ושהכתובת של `/webhook/vapi`
נגישה מהאינטרנט (הפלטפורמה שולחת אליה).

המערכת כבר יודעת לתרגם את פורמט ה-webhook של Vapi ו-Retell (ראה
`app/adapters.py`) למנוע הפנימי: היא מזהה מתקשר, מנתחת עם Claude, ורושמת שיחה,
משימה וחיוב.

## סדר עבודה מומלץ

1. לארח את ה-Backend ולוודא `/health` ו-`/webhook/vapi` נגישים.
2. להקים Assistant בפלטפורמה עם התסריט, Claude, וקול עברי, ולכוון את ה-webhook.
3. להפעיל העברת שיחות מ-03-3104959 למספר הפלטפורמה.
4. שיחת בדיקה: להתקשר ל-03-3104959 ולוודא שנרשמה שיחה במסד.

## הערה על עלות

בתשלום לפי שימוש: הפלטפורמה (Vapi/Retell), ספקי הקול/התמלול, ושימוש ב-Claude.
ההעברה עצמה היא הדבר שאתה כבר עושה.

## מסלול חלופי (LiveKit + Twilio)

אם בעתיד תעדיף שליטה מלאה על הקוד, `livekit_worker.py` הוא שלד לעובד LiveKit
שמתחבר לאותו Backend. המסלול הזה דורש יותר הקמה. כרגע מסלול הפלטפורמה המוכנה
מהיר יותר.
