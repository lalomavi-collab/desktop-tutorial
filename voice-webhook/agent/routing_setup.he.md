# ניתוב הקו 03-3104959 למענה ה-AI (קו VoIP ישראלי)

מדריך לחיבור מספר VoIP ישראלי (מ-019 / 012 / סלקום / פרטנר וכו') לסוכן הקול
מבוסס LiveKit. הרעיון: הספק שלך מעביר שיחות נכנסות של 03-3104959 אל LiveKit דרך
SIP, ו-LiveKit מפעיל את סוכן ה-AI. בסיום השיחה נשלח webhook ל-Backend שבנינו.

זרימה כללית:

```
מתקשר > 03-3104959 > ספק ה-VoIP שלך > (SIP) > LiveKit SIP
      > חדר LiveKit + סוכן AI (דיבור, Claude, הקראה)
      > בסיום: POST /webhook/livekit/ended > Supabase
```

---

## שלב א': מה לבקש מספק ה-VoIP

פנה לתמיכה של הספק (019 / 012 / סלקום / פרטנר / אחר) ובקש:

1. לאשר שהמספר 03-3104959 תומך ב-SIP trunk או בניתוב שיחות נכנסות ליעד SIP חיצוני.
2. לקבל את פרטי ה-SIP: כתובת השרת (SIP host / domain), פורט, ואם צריך שם משתמש
   וסיסמה, או האם הניתוב מבוסס כתובת IP.
3. לבקש שינתבו שיחות נכנסות של המספר אל כתובת ה-SIP של LiveKit (תתקבל בשלב ב').
4. לוודא שאין חסימה של יעדי SIP חיצוניים בחבילה שלך.

רשום את מה שקיבלת. בלי זה אי אפשר להתקדם בצד LiveKit.

## שלב ב': הגדרת LiveKit (הכי פשוט: LiveKit Cloud)

1. פתח חשבון ב-LiveKit Cloud (יש תוכנית התחלה). קבל `LIVEKIT_URL`,
   `LIVEKIT_API_KEY`, `LIVEKIT_API_SECRET`.
2. הפעל את רכיב ה-SIP של LiveKit. ב-Cloud הוא מובנה. תקבל כתובת SIP נכנסת
   (SIP host) של LiveKit, וזו הכתובת שתמסור לספק ה-VoIP בשלב א'(3).
3. צור Inbound Trunk: מגדיר מאילו מספרים/כתובות מותר לקבל שיחות (המספר או ה-IP
   של הספק שלך).
4. צור Dispatch Rule: כל שיחה נכנסת נכנסת לחדר ייעודי ומפעילה את סוכן ה-AI.

הפקודות המדויקות נמצאות בתיעוד LiveKit SIP. את שם הסוכן קובעים בכלל ה-dispatch.

## שלב ג': סוכן ה-AI (העובד)

השלד נמצא ב-`livekit_worker.py`. הוא:

1. עונה לשיחה ומדבר לפי התסריט הקבוע ב-`lalum_voice_agent.he.md`.
2. ממיר דיבור לטקסט (STT), שולח ל-Claude, ומקריא תשובה (TTS).
3. בסיום השיחה שולח `POST /webhook/livekit/ended` עם call_sid, caller_phone,
   duration_seconds ו-transcript.

צריך מפתחות: `ANTHROPIC_API_KEY`, וספקי STT/TTS. לעברית מומלץ לבחור ספק עם
תמיכה טובה בעברית (איכות הזיהוי וההקראה בעברית משתנה בין ספקים, כדאי לבדוק
דוגמה לפני שמתחייבים).

## שלב ד': חיבור ל-Backend

הגדר ב-סוכן משתנה סביבה `WEBHOOK_URL` שמצביע על הכתובת הציבורית של השירות שבנינו,
למשל `https://<host>/webhook/livekit/ended`. שם צריך גם `SUPABASE_URL`,
`SUPABASE_SERVICE_ROLE_KEY` ו-`ANTHROPIC_API_KEY` (שמות בלבד, בסביבת ההרצה).

## סדר עבודה מומלץ

1. לארח את שירות ה-Backend (voice-webhook) עם הסודות. לוודא ש-`/health` עונה.
2. לפתוח LiveKit Cloud ולהפעיל SIP (שלב ב').
3. לתת לספק ה-VoIP את כתובת ה-SIP של LiveKit ולבקש ניתוב (שלב א').
4. להשלים ולפרוס את העובד (שלב ג') עם מפתחות STT/TTS ו-Claude.
5. לבצע שיחת בדיקה ל-03-3104959 ולוודא שנרשמה שיחה במסד.

## הערה על עלות וזמן

החלקים בתשלום: חשבון LiveKit, ספקי STT/TTS, ושימוש ב-Claude. אלה עלויות לפי
שימוש. ההקמה עצמה טכנית וברת ביצוע, והחלק הכי תלוי-ספק הוא ניתוב ה-SIP (שלב א').
